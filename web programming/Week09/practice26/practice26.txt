처음에 이클립스에서 프로그래밍을 해서 run as로 돌려서 chrome이 떴을 때 exception이 하나 새로 만들어질 것. 
1번. 웹브라우저 열고 여기서 session.jsp 실행시키면 일단 session 하나 만들어질 것.
그 다음, 현재 열고 있는 웹브라우저에서 새 탭을 열면 session은 그대로 같이 간다. 
이클립스에 접속 후 로그인 한 다음 새 탭을 만든 후, 이클립스 다시 가면 아까 로그인한 정보 그대로 로그인되어있다.
그 다음, 3번으로 새 창을 만들어 보기로 했다. 
이클립스 다시 접속한 상태에서 새 창이라고 되어있는 걸 띄우면 로그인 세션 상태 그대로 유지 된다.
새로운 session이 만들어지는게 아닌 기존의 session이 계속 유지가 된다.
4번, chrome이 열려있는 상태에서 edge를 새로 띄우고 session_01.jsp를 실행시키면 다른 회사 웹브라우저는 별도의 프로그램이기 때문에 별도의 session이 만들어진다.
5번, 1-4번까지 띄웠던 모든 창을 다 닫고 chrome 새로 시작해 session_01.jsp 시작하면 session 새로 만들어질 것.
1-3번까지는 처음에 만들어졌던 session이 계속 사용될 것이고, 4-5번은 새로운 session이 만들어져야 한다.
